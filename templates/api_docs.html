API Documentation

Навигация
Аутентификация Товары Операции Аналитика
Аутентификация
Получение токена доступа
POST /api/v1/auth/login
Пример запроса: 
{
  "username": "admin",
  "password": "password123"
}
Пример ответа: 
{
  "access_token": "eyJ0eXAiOiJKV1QiLCJhbGciOiJIUzI1NiJ9...",
  "user": {
    "id": 1,
    "username": "admin",
    "email": "admin@warehouse.com",
    "role": "admin"
  }
}
Товары
Управление товарами
GET /api/v1/products

Получить список всех активных товаров

Заголовки: Authorization: Bearer <token>
GET /api/v1/products/<id>

Получить товар по ID

POST /api/v1/products

Создать новый товар (только для администраторов)

Пример запроса: 
{
  "name": "Новый товар",
  "description": "Описание товара",
  "unit": "шт",
  "price": 100.0,
  "cost_price": 80.0,
  "min_stock": 5,
  "max_stock": 1000,
  "category_id": 1,
  "supplier_id": 1
}
Операции
Складские операции
POST /api/v1/operations

Создать складскую операцию (только для кладовщиков и администраторов)

Пример запроса: 
{
  "type": "receipt",
  "product_id": 1,
  "quantity": 10,
  "document_number": "RECEIPT-001",
  "notes": "Поступление от поставщика"
}
Типы операций:
  • receipt - Приём товара
  • shipment - Отгрузка товара
Аналитика
Статистика и отчёты
GET /api/v1/analytics/stock

Получить аналитику по складу

В ответе содержится:
  • Общее количество товаров
  • Общая стоимость запасов
  • Количество товаров с низким запасом
  • Количество отсутствующих товаров
  • Аналитика по категориям
Тестирование API

Для тестирования API можно использовать Postman, curl или любой другой HTTP клиент.

Пример с curl:
Пример использования: 
# Получить токен аутентификации
curl -X POST http://localhost:5000/api/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123"}'

# Получить список товаров
curl -X GET http://localhost:5000/api/v1/products \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
